# План реализации: PacksByUsers компонент с таблицей пакетов

**Дата**: 04.04.2026  
**Статус**: На утверждение → В разработке → ✅ Готово

## TL;DR

Создать компонент таблицы в `PacksByUsers`, где:
- **Строки** — дни похода
- **Столбцы** — пакеты участников (по номерам)
- **Содержимое ячейки** — общий вес пакета, названия продуктов (компактно), количество товаров

Использовать CSS Grid для удобства интеграции drag-and-drop позже.

## Требования

- ✅ Содержимое ячейки: вес пакета + названия продуктов + тотальное количество (компактно)
- ✅ Grid layout для удобства реализации drag-and-drop
- ✅ Заголовки столбцов: номера пакетов (1, 2, 3...)
- ✅ Без перетягивания на первом этапе
- ✅ Код готов к добавлению drag-and-drop (структура и комментарии)

   - Тип для промежуточного представления `PacksByDayData`:
     ```

       packNumber: number,
     ```

2. **Добавить helper функции** в `src/components/hiking-page/hiking-helpers.ts`

4. **Создать подкомпонент `PacksHeader`**

5. **Создать подкомпонент `PacksRow`**

## Декайдированы решения

- **Grid вместо `<table>`**: легче интегрировать drag-and-drop, более гибкая разметка, соответствует стилю кодовой базы (PacksByDays тоже использует div-based grid)
- **Группировка по дням, столбцы = пакеты**: интуитивное отображение ("какие товары у каждого участника в какой день")
- **Вес + имена + количество (компактно)**: достаточно информации для планирования без перегруженности
- **Helper функции в `hiking-helpers.ts`**: переиспользуемость, облегчает тестирование, разделение логики от UI
- **Подготовка структуры к drag-and-drop**: data-атрибуты и обертки позволят интегрировать @dnd-kit без переписывания компонентов
